🔧 导出功能修复报告

服务标签位置偏移和DOM导出图片模糊问题修复

🎯 服务标签位置修复 已修复

  • 问题描述：Canvas导出时服务标签位置偏移，与预览不一致
  • 根本原因：getElementActualPosition函数未考虑变换矩阵和父元素偏移
  • 修复方案：
    • 添加变换矩阵计算支持
    • 特殊处理服务标签的父元素位置
    • 根据文本对齐方式调整坐标
    • 微调垂直位置以匹配DOM渲染

📸 DOM导出图片质量提升 已优化

  • 问题描述：DOM导出的图片模糊，清晰度不够
  • 根本原因：html2canvas缩放比例过低，未充分利用设备像素比
  • 优化方案：
    • 将缩放比例从2倍提升到4倍
    • 考虑设备像素比进行动态调整
    • 优化Canvas尺寸计算
    • 保持所有导出方法的一致性

🔧 技术实现细节

1. 服务标签位置计算优化

// 改进的位置计算函数 const getElementActualPosition = (selector: string) => { const element = cardElement?.querySelector(selector) as HTMLElement if (!element) return null const rect = element.getBoundingClientRect() const cardRect = cardElement!.getBoundingClientRect() const computedStyle = window.getComputedStyle(element) // 获取变换矩阵以计算实际位置 const transform = computedStyle.transform let actualX = rect.left - cardRect.left let actualY = rect.top - cardRect.top // 如果有变换，需要调整位置 if (transform && transform !== 'none') { const matrix = new DOMMatrix(transform) actualX += matrix.m41 actualY += matrix.m42 } // 对于服务标签，需要特殊处理位置偏移 if (selector.includes('Label')) { const parentElement = element.parentElement if (parentElement) { const parentRect = parentElement.getBoundingClientRect() const parentCardRect = cardElement!.getBoundingClientRect() actualX = parentRect.left - parentCardRect.left actualY = parentRect.top - parentCardRect.top } } return { x: actualX, y: actualY, ... } }
